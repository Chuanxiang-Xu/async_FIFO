This page collects the waveform-led learning path for the project. It uses
checked-in assets and reproducible testbenches only; there are no fabricated

## How to Regenerate

The tutorial waveform VCD is generated by:

```bash
make tutorial
```

The output is:

```text
build/tutorial_async_fifo.vcd
```

Open the VCD in your waveform viewer and inspect the same signals shown in
the static SVG:

```text
wr_clk, wr_rstn, wr_en, wr_data, full
rd_clk, rd_rstn, rd_en, rd_data, rd_valid, empty
```

## Standard Async FIFO Timing

![Representative async FIFO waveform](assets/async_fifo_waveform.svg)

What to notice:

- write and read clocks are unrelated;
- accepted writes happen only when `wr_en && !full`;
- the read side sees data only after the write Gray pointer synchronizes over;
- `rd_valid` qualifies returned `rd_data`;
- `empty` deassertion is conservative, not instantaneous.

## Tutorial Fill and Drain Scenario

![Tutorial async FIFO waveform](assets/tutorial_waveform.svg)

This is the concrete depth-4 scenario from `test/tb_fifo_tutorial.sv`:

| Moment | Lesson |
|---|---|
| Writes `A0`, `A1`, `A2`, `A3` | Write requests are accepted while `full=0`. |
| `empty` deasserts later | The read domain waits for synchronized write-pointer visibility. |
| `full` asserts | The write domain predicts the next write would overwrite unread data. |
| Attempted `EE` writes while full | Blocked writes do not enter storage. |
| Reads return `A0`, then `A1` | FIFO ordering is visible through `rd_valid`. |
| `full` deasserts later | Freed space is visible after the read pointer synchronizes back. |

## Standard Read vs FWFT Read

![Standard read versus FWFT read timing](assets/fwft_vs_standard_waveform.svg)

The standard FIFO is request/response:

```text
rd_en && !empty  requests a read
rd_valid         marks returned rd_data
```

The FWFT wrapper is observe/consume:

```text
rd_valid == 1       rd_data is already visible
rd_en && rd_valid   consumes the visible word
empty == !rd_valid
```

## Architecture View

![Async FIFO core and wrapper architecture](assets/architecture.svg)

Use this view to connect waveforms back to implementation ownership:

- the equal-width core owns pointer, synchronizer, flag, RAM, and `rd_valid`
  behavior;
- FWFT, width conversion, stream, bidirectional, and RAMIF modules are
  wrappers around the CDC core or its pointer-control pattern;
- payload data crosses through RAM, not through per-bit synchronizers.
