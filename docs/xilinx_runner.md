# Xilinx Self-Hosted Runner

The optional Xilinx CI job runs the real Vivado CDC elaboration checks on a
licensed Linux machine. GitHub-hosted runners do not include Vivado.

## Required runner state

- Linux x86-64;
- Vivado 2025.2 available to the runner service account;
- a valid synthesis license for `xc7z020clg400-1`;
- GNU Make and Bash;
- runner labels `self-hosted`, `linux`, `x64`, and `vivado-2025.2`.

Register the machine from:

```text
Repository Settings -> Actions -> Runners -> New self-hosted runner
```

Use the temporary URL and token generated by GitHub. A representative
registration command is:

```sh
./config.sh \
  --url https://github.com/OWNER/REPOSITORY \
  --token TEMPORARY_TOKEN \
  --name vivado-2025-2 \
  --labels vivado-2025.2 \
  --work _work
```

The GitHub-provided token and repository URL must be substituted at
registration time; they cannot be stored safely in this repository.

## Service environment

The service account must see Vivado without relying on an interactive shell
startup file. Either put Vivado in the service `PATH`, or set an absolute
workflow environment variable:

```text
VIVADO=/opt/amd/Vivado/2025.2/bin/vivado
```

Before enabling CI, run:

```sh
scripts/check_xilinx_runner.sh
```

This fails unless the expected Vivado version is available and both the
single-instance and multi-instance positive/negative CDC tests pass.

Finally create repository Actions variable:

```text
XILINX_CI_ENABLED=true
```

The job runs on pushes, internal pull requests, and manual workflow dispatch.
Fork pull requests are intentionally excluded from the self-hosted machine.

For a required branch-protection check, add `xilinx-vivado-cdc` to the
repository's required status checks only after the runner is online.
